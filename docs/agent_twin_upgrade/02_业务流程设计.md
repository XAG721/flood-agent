# AgentTwin Flood V3 业务流程设计

## 1. 设计原则

1. 风险判断尽量确定化，智能体输出尽量结构化。
2. 高风险动作必须经过审批，不允许模型越权执行。
3. 业务流程应兼容现有 `V2` 的事件、proposal、通知和审计主链路。
4. 前端流程应统一围绕“数字孪生主屏 + 智能体对话控制 + 审批闭环”展开。

## 2. 总体业务闭环

```text
天气预警/人工输入
-> 事件解析
-> 影响链生成
-> 行动方案推演
-> 多智能体会商
-> 审计校验
-> proposal 生成
-> 人工审批
-> 分众预警
-> 执行留痕
-> 事后复盘
```

对应前端的主交互闭环建议表达为：

```text
总览大屏态势查看
-> 选择重点对象
-> 唤起智能体对话控制
-> 追问与推演
-> 查看待审批 proposal
-> 审批动作
-> 输出分众预警
-> 进入审计与复盘
```

## 3. 关键流程

## 3.1 事件创建与解析流程

1. 值班员录入天气文本，或系统接收外部预警文本。
2. `WeatherEventParser` 解析灾害类型、区域、时窗和等级。
3. 系统生成结构化 `weather_event`。
4. 若解析失败，则进入人工补录。

参与方：

- 值班员
- WeatherAgent
- 事件服务

## 3.2 影响链生成流程

1. 系统读取事件、区域画像、对象画像、历史案例和 SOP。
2. `ImpactChainEngine` 识别受影响对象。
3. 系统生成主要影响、次生影响、证据链和置信度。
4. 输出结构化 `impact_chains`。

参与方：

- ImpactAgent
- RAG 服务
- 对象画像库
- 案例库

## 3.3 候选行动推演流程

1. 基于影响链匹配动作模板。
2. 生成多个候选行动方案。
3. `ActionScorer` 从时效、资源、脆弱群体保护、审批复杂度等维度评分。
4. 输出排序结果与不推荐原因。

参与方：

- ActionAgent
- ResourceAgent
- 动作模板库

## 3.4 多智能体会商流程

1. Orchestrator 向各角色智能体分发同一事件上下文。
2. 各智能体输出自己的结构化结论。
3. AuditAgent 检查地名、资源、权限、证据和冲突。
4. Orchestrator 汇总形成最终建议和待审批 proposal。

参与方：

- Orchestrator
- WeatherAgent
- ImpactAgent
- ActionAgent
- ResourceAgent
- CommunicationAgent
- AuditAgent

## 3.5 proposal 审批流程

1. 系统生成 proposal 并进入待审批队列。
2. 前端在总览大屏右侧决策带或行动处置页提示待审批事项。
3. 指挥员查看建议、证据、对象、责任部门和时限。
4. 指挥员批准、驳回或修改参数范围。
5. 审批结果写入审计与执行日志。

参与方：

- 指挥员
- 审批策略服务
- 审计服务

## 3.6 分众预警发布流程

1. `AudienceWarningCenter` 基于已审批 proposal 生成统一 Alert Object。
2. 系统按受众改写为领导版、部门版、社区版、公众版。
3. `ConsistencyChecker` 校验口径一致性。
4. 审批通过后进入通知中心。

## 3.7 复盘学习流程

1. 事件结束后生成高风险复盘。
2. 抽取有效动作、失败动作、协同缺口和复用规则。
3. 更新知识图谱、动作模板库、案例库和 Prompt 经验。

## 4. 前端视角流程

## 4.1 总览大屏流程

1. 用户进入数字孪生城市洪涝态势主屏。
2. 系统默认展示当前事件、风险热区、重点对象和智能体摘要。
3. 用户点击某个重点对象。
4. 主屏联动更新对象详情、影响链摘要和建议行动。

## 4.2 智能体对话控制流程

1. 用户通过全局入口或对象快捷入口唤起智能体对话控制台。
2. 对话框自动带入当前事件与对象上下文。
3. 用户继续追问风险原因、证据、方案取舍和审批建议。
4. 智能体返回结构化解释、建议动作和下一步追问提示。

## 4.3 审批闭环流程

1. 用户在主屏或行动处置页看到待审批 proposal。
2. 用户查看 proposal 详情与证据链。
3. 用户执行批准、驳回或暂缓。
4. 系统回写主屏状态并更新通知链路。

## 5. 参与方视角

| 参与方 | 主要职责 |
|---|---|
| 系统 | 触发流程、保存数据、推送待办 |
| 值班员 | 创建事件、补录信息、查看初始结果 |
| 指挥员 | 审批 proposal、决定高风险动作 |
| 部门人员 | 接收任务单、回传执行情况 |
| 社区人员 | 执行巡查、居民提醒、反馈现场情况 |
| 智能体 | 生成结构化分析、解释和建议 |

## 6. 与现有 V2 的衔接

当前 `V2` 已具备：

- 事件入口
- 风险更新
- proposal 待审批队列
- SSE 推送
- 审计、通知、执行日志

`V3` 的接入建议为：

1. 保留 `V2` 事件和 proposal 主链路。
2. 在 proposal 生成前插入“影响链 + 推演 + 会商”步骤。
3. 在 proposal 批准后插入“分众预警”步骤。
4. 在前端侧把原有页面升级为数字孪生主屏和智能体对话控制闭环。

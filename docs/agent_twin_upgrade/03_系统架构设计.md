# AgentTwin Flood V3 系统架构设计

## 1. 架构目标

在不推翻当前 `V2` 单体原型的前提下，引入一层 `V3 AgentTwin Decision Layer`，把现有系统从“风险结果驱动”升级为“影响链与行动推演驱动”。

## 2. 当前基线

当前代码基线主要由以下模块构成：

- 接口层：`flood_system/api.py`
- 应用装配：`flood_system/system.py`
- 数据存储：`flood_system/repository.py`
- `V2` 业务层：`flood_system/v2/`
- 前端控制台：`frontend/src/App.tsx`
- 前端状态编排：`frontend/src/hooks/useV2OperatorConsole.ts`

## 3. 目标架构

```text
前端展示层
-> 业务接口层
-> V3 AgentTwin 决策层
-> V2 平台能力复用层
-> 数据与知识层
-> 治理与审计层
```

## 4. 分层说明

## 4.1 前端展示层

职责：

- 展示影响态势、影响链、会商结果、候选方案和分众预警
- 承担审批、筛选和证据查看

基于当前前端的承接点：

- `frontend/src/App.tsx`
- `frontend/src/components/GlobalRegionalProposalDialog.tsx`
- `frontend/src/pages/OverviewPage.tsx`
- `frontend/src/pages/OperationsPage.tsx`
- `frontend/src/pages/AgentsPage.tsx`

## 4.2 业务接口层

职责：

- 保留 `/v2/*`
- 新增 `/api/v3/*`
- 提供 REST + SSE/WebSocket

建议新增：

- `POST /api/v3/weather-events/parse`
- `POST /api/v3/events/{event_id}/impact-chains/generate`
- `POST /api/v3/events/{event_id}/scenarios/generate`
- `POST /api/v3/events/{event_id}/agent-council/run`
- `POST /api/v3/proposals/{proposal_id}/warnings/generate`

## 4.3 V3 AgentTwin 决策层

建议新增目录：

```text
flood_system/v3/
  weather_event_parser.py
  impact_ontology.py
  impact_graph.py
  impact_chain_engine.py
  action_template_store.py
  scenario_decision_lab.py
  action_scorer.py
  agent_roles.py
  agent_council.py
  agent_memory.py
  evidence_verifier.py
  audience_warning_center.py
  cap_like_alert.py
  message_consistency_checker.py
  approval_policy.py
  decision_memory.py
```

职责：

- 事件解析
- 影响链生成
- 多方案推演
- 多智能体会商
- 分众预警
- 审批边界控制

## 4.4 V2 平台能力复用层

复用模块：

- `regional_proposals.py`
- `notification_gateway.py`
- `memory_store.py`
- `reporting.py`
- `security.py`

复用原则：

- `V3` 产出结构化 proposal
- 仍通过现有 proposal、通知、审计和执行日志链路落库

## 4.5 数据与知识层

现状：

- SQLite 作为主存储
- `rag_raw/` 作为文档知识源
- `data_sources/` 作为原始与标准化数据

升级后新增重点：

- 影响链数据
- 动作模板库
- 智能体会商记录
- 分众预警记录
- 复盘经验库

## 4.6 治理与审计层

复用当前治理基础：

- `flood_system/v2/security.py`
- `flood_system/v2/multi_agent.py` 中的 supervisor、trigger、timeline、audit
- `flood_system/repository.py` 中的 `v2_audit_records`、`v2_operational_alerts`

新增目标：

- 校验证据引用
- 校验消息一致性
- 校验审批越权
- 记录模型版本、提示词版本和工具调用

## 5. 调用关系

### 5.1 主链路

1. `weather_event_parser`
2. `impact_chain_engine`
3. `scenario_decision_lab`
4. `agent_council`
5. `evidence_verifier`
6. `regional_proposals / notification_gateway`
7. `audience_warning_center`

### 5.2 后台链路

1. SupervisorLoop 继续负责后台调度
2. Housekeeping 继续负责归档
3. Reporting 继续负责日报和复盘

## 6. 部署建议

### 6.1 开发阶段

- 单体 FastAPI + SQLite
- React 前端本地开发
- 3D 端保留独立部署

### 6.2 后续可扩展

- SQLite -> PostgreSQL
- TriggerEvent -> Redis/Kafka
- SSE -> WebSocket
- RAG 文档 -> 向量库

## 7. 兼容策略

1. 不删除现有 `/v2/*` 接口。
2. `V3` 先以补充接口和补充表方式接入。
3. 前端先增加新面板，不强制重写现有 `V2` 控制台。

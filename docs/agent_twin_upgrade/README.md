# AgentTwin Flood 升级文档包

## 1. 文档定位

本目录用于承接 [`AgentTwin-Flood-Requirements.md`](../../AgentTwin-Flood-Requirements.md) 中提出的升级需求，形成一套面向 `V3 / AgentTwin` 的设计文档包。

本目录中的文档不替代 `docs/v2_baseline/` 中的现状文档，而是用于描述下一阶段的目标形态。

统一口径如下：

- 后端定位：`AgentTwin Flood V3` 辅助决策系统
- 前端定位：**数字孪生智能体洪水预警系统前端**
- 升级主线：影响链、行动推演、智能体协同、人工审批、分众预警、审计复盘

## 2. 与当前代码的关系

当前仓库的事实基线如下：

- 后端主入口：`flood_system/api.py`
- 应用装配入口：`flood_system/system.py`
- 运行时存储：`flood_system/repository.py`
- `V2` 业务实现：`flood_system/v2/`
- 前端入口：`frontend/src/App.tsx`
- 前端状态编排：`frontend/src/hooks/useV2OperatorConsole.ts`

因此，本目录采用“`V2` 已实现基线 + `V3` 升级目标”的写法，避免文档设计脱离现有仓库现实。

## 5. 前端统一口径

关于前端，所有相关文档统一采用以下表达：

1. 前端不是普通业务后台，而是“数字孪生智能体洪水预警系统前端”。
2. 总览页不是信息看板，而是“数字孪生城市洪涝态势主屏”。
3. 智能体交互不是普通聊天，而是“智能体对话控制”或“智能体指挥与追问控制台”。
4. 主交互链路应围绕“主屏态势感知 -> 对象联动 -> 智能体追问 -> proposal 审批 -> 分众预警”展开。

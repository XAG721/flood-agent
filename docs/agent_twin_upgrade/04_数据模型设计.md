# AgentTwin Flood AgentTwin 数据模型设计

## 1. 设计原则

1. 保留现有 `v2_*` 表作为运行基线。
2. `AgentTwin` 新增表优先围绕“影响链、会商、分众预警、复盘经验”建设。
3. 结构化字段优先，复杂内容用 `json` 扩展。

## 2. 当前基线表

当前 `flood_system/repository.py` 中已存在的关键表包括：

- `v2_events`
- `v2_simulation_updates`
- `v2_hazard_states`
- `v2_entity_profiles`
- `v2_action_proposals`
- `v2_notification_drafts`
- `v2_execution_logs`
- `v2_agent_tasks`
- `v2_agent_results`
- `v2_trigger_events`
- `v2_audit_records`
- `v2_experience_records`
- `v2_long_term_memories`

这意味着当前系统已经具备：

- 事件
- proposal
- 多智能体任务
- 审计
- 经验沉淀

等核心骨架。

## 3. AgentTwin 建议新增表

## 3.1 `v3_weather_events`

| 字段 | 类型 | 说明 |
|---|---|---|
| `weather_event_id` | TEXT PK | 天气事件 ID |
| `event_id` | TEXT FK | 关联主事件 |
| `source_text` | TEXT | 原始文本 |
| `hazard_type` | TEXT | 灾害类型 |
| `warning_level` | TEXT | 预警等级 |
| `certainty` | TEXT | 可能/较高/确定 |
| `affected_area_json` | TEXT | 影响区域 |
| `time_window_json` | TEXT | 时间窗口 |
| `parsed_by` | TEXT | 解析来源 |
| `created_at` | TEXT | 创建时间 |

## 4. 主外键关系

```text
v2_events
  -> v3_weather_events
  -> v3_impact_chains
  -> v3_scenarios
  -> v3_agent_deliberations

v2_action_proposals
  -> v3_audience_warnings

v2_entity_profiles
  -> v3_impact_objects
  -> v3_impact_chains
```

## 5. 索引建议

- `v3_impact_chains(event_id, object_id, created_at)`
- `v3_scenarios(event_id, score DESC)`
- `v3_agent_deliberations(event_id, agent_name, created_at)`
- `v3_audience_warnings(proposal_id, audience_type, created_at)`

## 6. 与现有模型的映射建议

- `ActionProposal` 继续作为最终待审批实体
- `NotificationDraft` 继续作为批准后的通知草稿实体
- `ExecutionLogEntry` 继续作为执行日志实体
- `v3_*` 负责补充“生成前”的分析与“发布前”的分众内容

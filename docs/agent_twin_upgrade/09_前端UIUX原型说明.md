# AgentTwin Flood 前端 UI/UX 原型说明

## 1. 定位

本前端不是毕业设计展示页，也不是普通后台管理系统，而是面向甲方演示和联调验证的 **数字孪生智能体洪水预警生产级 demo 前端**。

前端目标是让甲方在第一屏就理解三件事：

- 系统有真实数字孪生空间表达能力。
- 系统能把洪水事件转成对象级影响链和行动建议。
- 系统通过智能体会商、人工审批和分众预警形成可追溯闭环。

## 2. 设计原则

### 2.1 展示优先

当前阶段前端主要承担演示价值，必须优先保证视觉冲击力、操作连续性和讲解清晰度。

- 首页是演示主舞台。
- Cesium 城市场景是第一视觉锚点。
- 页面信息密度要高，但重点必须清楚。
- 所有核心状态都要服务“甲方能听懂、能看见、能操作”。

### 2.2 工程可信

虽然是 demo，但必须具备生产级测试品的工程边界。

- 接口以 `/v3/*` 聚合读模型为主。
- 审批、通知和审计继续复用 `/v2/*` 稳定闭环。
- LLM、SSE、Cesium 或数据失败时要有降级态。
- 高风险动作必须保留人工审批闸门。

### 2.3 不做普通后台

禁止把首页做成指标卡片堆叠。

首页应是“数字孪生指挥台”，而不是“数据看板”。二级页可以承载更多管理和细节，但整体风格仍要服务智能体洪水预警系统的产品定位。

## 3. 首页原型

首页 `/` 定义为 `DigitalTwinImpactScreen`，是整个 demo 的核心演示入口。

### 3.1 首屏结构

推荐结构：

```text
顶部产品态势区
  - 当前事件
  - 风险等级
  - V3/SSE/Agent 状态
  - 关键操作入口

生产级 demo proof strip
  - 3D twin
  - Agent council
  - Human gate
  - Audience warnings

左侧态势带
  - 重点对象
  - 实时信号
  - 风险解释

中央数字孪生画布
  - Cesium 城市场景
  - 对象点位
  - proposal/warning 状态标识
  - hover spotlight
  - 空间闭环清单

右侧指挥台
  - 当前焦点对象
  - 推荐动作矩阵
  - 证据阶梯
  - proposal 审批与 warning 生成微流程
```

### 3.2 必须可演示的交互

- 点击左侧对象，中央地图聚焦，右侧指挥台同步切换。
- 点击 Cesium 点位，焦点对象、证据、建议动作同步变化。
- hover 地图对象时显示对象级 spotlight。
- 在右侧指挥台直接选择待审批 proposal。
- 在右侧指挥台填写审批意见并批准或驳回。
- 批准后直接生成分众 warning drafts。
- 打开智能体对话抽屉，基于当前对象继续追问。

### 3.3 展示语言

首页文案要短、直接、面向操作：

- 使用 `Production demo mode` 表示当前是生产级测试品演示态。
- 使用 `Human gate` 表示高风险动作仍需要人工放行。
- 使用 `Audience warnings` 表示分众预警结果。
- 使用 `Spatial closure layer` 表示地图中的 proposal/warning 状态联动。

## 4. 智能体对话控制

智能体对话不是普通聊天，而是“智能体指挥与追问控制台”。

它应承担：

- 解释当前对象为什么高风险。
- 展示证据、影响链和风险提醒。
- 给出推荐动作和后续追问。
- 触发 proposal 生成。
- 引导进入审批和 warning 闭环。

抽屉结构：

```text
顶部：当前对象 / 事件 / 队列状态
左侧：对话历史、回答、证据、推荐动作
右侧：影响链摘要、追问建议、proposal/closure 入口
底部：输入框、建议 prompt、发送按钮
```

## 5. `/agents` 会商室

`/agents` 不应只是调试后台，而应作为“多智能体会商室”展示。

核心展示内容：

- Agent Council 摘要
- 角色差异与分歧
- 证据对照板
- Supervisor 编排路径
- Open questions
- Blocked by
- 会商到 proposal/warning 闭环的推进状态

该页面用于向甲方说明：系统不是简单调用大模型，而是有角色分工、证据约束、编排逻辑和审计边界。

## 6. `/operations` 行动处置台

`/operations` 承接更完整的方案处置和执行视图。

它应展示：

- proposal 队列
- 区域分析包
- 审批历史
- 分众 warning drafts
- 工具调用和执行链路
- 当前对象与事件的处置时间线

首页已经具备轻量审批微流程，`/operations` 则用于承载更详细的批量处理、复核和执行编排。

## 7. `/reliability` 审计与可靠性中心

`/reliability` 面向工程可信和甲方风险顾虑。

它应展示：

- Agent council audit
- 审计记录
- 权限边界
- SSE / 模型 / 数据状态
- proposal -> warning -> audit 的闭环结果
- 可复盘输出

该页面要回答“系统为什么可信、出了问题如何降级、谁批准了什么”。

## 8. 视觉风格

视觉方向：城市级数字孪生指挥台。

关键词：

- 深色空间底
- Cesium 城市模型
- 高对比对象标识
- 暖色行动强调
- 冷色系统状态
- 克制的 HUD
- 面向大屏展示的排版

不建议：

- 白底管理后台风格
- 大量普通卡片堆叠
- 过多渐变和装饰
- 只展示数字，不展示空间关系

## 9. 动效建议

动效服务演示，不做装饰。

- 首页进入时，主标题和 proof strip 分层进入。
- 对象 hover 时显示 spotlight。
- 对象选中时地图点位和右侧指挥台同步变化。
- 智能体对话抽屉使用平滑侧滑。
- proposal 状态变化后，地图状态层和右侧闭环区同步刷新。

## 10. 验收标准

面向生产级 demo 的前端验收标准：

- 首页能作为甲方演示主屏独立讲清楚系统价值。
- 首页能完成对象选择、智能体追问、proposal 审批、warning 生成。
- `/agents` 能解释多智能体会商差异和编排逻辑。
- `/operations` 能承载详细处置和执行视图。
- `/reliability` 能说明审计、权限和降级策略。
- Cesium 或 LLM 异常时不白屏，仍能展示降级信息。
- 视觉上明显区别于普通后台，具备数字孪生指挥台观感。

# 甲方演示脚本

## 1. 演示定位

本脚本用于展示 `AgentTwin Flood` 作为生产级 demo 测试品的核心能力。

演示目标不是讲解所有后台配置，而是在 8-12 分钟内让甲方清楚看到：

- 系统能把城市洪涝态势投射到数字孪生主屏
- 系统能定位重点对象并解释影响链
- 多智能体会商不是黑盒，而是能展示角色差异、证据与审计边界
- 高风险动作进入人工审批闸门
- 审批后可以生成分众 warning，并留下审计与执行闭环
- 即使 LLM、SSE 或 3D 资源降级，系统仍能保留稳定演示态

## 2. 演示前准备

### 2.0 推荐一键启动

```powershell
.\scripts\start-demo.ps1
```

该脚本会自动完成：

- 重建演示主库
- 检查演示主库
- 使用 `FLOOD_DB_PATH` 指向演示库启动后端
- 启动前端开发服务
- 打开首页

如需保留现有数据库，可使用：

```powershell
.\scripts\start-demo.ps1 -SkipRebuild
```

### 2.1 重建演示主库

```powershell
C:\Users\Administrator\anaconda3\python.exe scripts\rebuild_demo_db.py --force
```

默认生成：

```text
data/flood_warning_system_demo.db
```

### 2.2 检查演示主库

```powershell
C:\Users\Administrator\anaconda3\python.exe scripts\inspect_demo_db.py
```

所有检查项都应为 `PASS`。

### 2.3 指定后端数据库

```powershell
$env:FLOOD_DB_PATH="D:\graduation_project\data\flood_warning_system_demo.db"
```

### 2.4 启动后端与前端

后端按项目现有启动方式运行，前端进入 `frontend` 后启动：

```powershell
npm.cmd run dev
```

## 3. 主演示事件

演示主事件固定为：

```text
event_demo_beilin_primary
```

事件名称：

```text
碑林区强降雨内涝风险演示事件
```

演示重点对象：

- 文艺路小学
- 碑林中心医院
- 和平门医养中心

- 建设路低洼网格

## 4. 演示主线

### Step 1：首页展示数字孪生主屏

打开首页 `/`。

讲解要点：

- 顶部客户价值摘要先说明当前事件、最危险对象和建议动作
- 左侧态势带展示重点对象和实时信号
- 中央 Cesium 画布展示对象空间位置、风险热区、联动路径和 proposal / warning 状态
- 右侧指挥台展示影响窗口、证据、推荐动作和闭环状态
- 可点击 `Command flythrough` 进入指挥巡航视角，自动聚焦主风险对象和闭环对象

建议话术：

```text
这里不是普通的风险后台，而是把事件、对象、空间位置、智能体会商、人工审批和预警闭环放到同一张指挥主屏里。
```

### Step 2：选择重点对象

点击左侧 `建设路低洼网格三组` 或地图对应点位。

讲解要点：

- 地图、左侧对象、右侧指挥台会同步联动
- 风险颜色、扩散圈和状态脉冲会跟随对象风险变化
- 右侧能看到影响窗口、风险提醒和推荐动作

建议话术：

```text
对象不是孤立列表项，选中以后会同时驱动空间画布、行动建议、证据和后续审批入口。
```

### Step 3：打开智能体对话控制台

点击 `打开智能体控制台`。

讲解要点：

- 对话不是普通聊天，而是带对象上下文的会商控制台
- 可以追问影响链、证据来源、推荐动作和 proposal 依据
- 回答中保留结构化证据、推荐动作和 proposal 入口

建议话术：

```text
智能体回答不会只给一段自由文本，而是把影响链、证据、建议动作和下一步入口拆开，让指挥员可以追问和接管。
```

### Step 4：展示主屏内审批微流程

在右侧 `Closure` 区查看待审批 proposal。

演示操作：

- 选择 `建设路低洼网格地下室回流协助与排涝封控`
- 输入审批意见
- 可以选择批准或驳回

讲解要点：

- 高风险动作不会自动执行
- proposal 进入人工审批闸门
- 审批意见会成为闭环记录的一部分

建议话术：

```text
这里体现的是人机协同边界：智能体负责生成方案和证据，指挥员负责最终放行。
```

### Step 5：展示 warning 生成闭环

查看已批准的 `碑林中心医院急诊与后勤入口保障` proposal。

讲解要点：

- 已批准动作可以直接生成分众 warning
- 演示库中预置了领导版、部门版、社区版和公众版 warning
- warning 与 proposal_id 关联，后续可审计和复盘

建议话术：

```text
系统不是停在生成方案，而是继续把方案转成不同受众能执行的预警内容。
```

### Step 6：进入 `/agents` 展示会商室

切换到 `/agents`。

讲解要点：

- 展示不同 agent 的角色判断
- 展示证据对照、缺口、交接建议
- 展示 supervisor 如何把不同结果编排成最终路径

建议话术：

```text
这页用来回答“智能体为什么这么建议”。不同角色的判断、证据和分歧都可以展开看。
```

### Step 7：进入 `/operations` 展示处置台

切换到 `/operations`。

讲解要点：

- 展示 proposal 队列
- 展示批准后的处置动作
- 展示 warning 草稿和执行痕迹

建议话术：

```text
处置台承接主屏产生的动作，让值班人员能继续处理、跟踪和复盘。
```

### Step 8：进入 `/reliability` 展示可信性

切换到 `/reliability`。

讲解要点：

- 展示审计结论
- 展示人工审批边界
- 展示模型和实时链路状态
- 展示闭环结果和执行留痕

建议话术：

```text
可靠性页回答的是“这个系统为什么可信、失败时怎么降级、动作怎么追溯”。
```

## 5. 降级演示口径

如果现场出现 LLM、SSE 或 3D 资源异常，不需要中断演示。

### 5.1 LLM 降级

讲解口径：

```text
模型不可用时，系统会回到结构化保守研判，不会把页面变成空白，也不会自动放行高风险动作。
```

### 5.2 SSE 降级

讲解口径：

```text
实时事件流恢复前，主屏会保留最近一次稳定快照，值班员仍然可以查看对象、proposal 和 warning 状态。
```

### 5.3 Cesium 降级

讲解口径：

```text
3D 资源加载失败时，系统仍保留对象列表、指挥台和闭环微流程，核心业务链路不依赖单一渲染资源。
```

## 6. 演示结束语

建议收束到这句话：

```text
AgentTwin Flood 的价值不只是生成一次风险判断，而是把数字孪生态势、智能体会商、人工审批、分众预警和审计追溯放进一条可运行的闭环里。
```

## 7. 演示验收清单

演示前确认：

- 演示库已重建
- `inspect_demo_db.py` 全部通过
- 后端使用 `FLOOD_DB_PATH` 指向演示库
- 首页可加载数字孪生主屏
- `Command flythrough` 可触发三维指挥巡航
- 至少有 `1` 条 pending proposal
- 至少有 `1` 条 approved proposal
- 至少有 `2` 条 warning
- `/agents` 可展示会商内容
- `/operations` 可展示 proposal 和 warning
- `/reliability` 可展示审计和闭环结果

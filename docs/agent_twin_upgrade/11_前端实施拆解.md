# AgentTwin Flood V3 前端实施拆解

## 1. 文档目标

本文件用于把前端设计文档进一步拆解为可执行的开发工作项，服务对象是前端开发、联调和阶段性验收。

本文件承接以下统一定位：

**数字孪生智能体洪水预警系统前端**

## 2. 实施原则

1. 优先复用现有 `App.tsx` 和 `useV2OperatorConsole.ts` 的数据链路。
2. 允许大幅重构页面骨架和视觉组织方式。
3. 优先完成“总览大屏 + 智能体对话控制 + proposal 审批入口”。
4. 尽量先打通主链路，再逐步增强细节表现和动效。

## 3. 页面拆解

## 3.1 P0：总览大屏

目标：

- 把首页重构为“数字孪生城市洪涝态势主屏”。

建议拆分模块：

- 顶部指挥条
- 左侧态势带
- 中央城市画布
- 右侧决策带
- 底部对象带

依赖数据：

- 当前事件
- 风险等级与趋势
- hazard state
- top impacts
- pending proposals
- agent status

验收标准：

- 首页不再是普通卡片布局
- 用户能在首页直接感知事件、对象和风险空间关系

## 3.2 P0：智能体对话控制弹窗

目标：

- 在不离开主屏的情况下完成智能体追问、解释和建议查看。

建议拆分模块：

- 弹窗标题区
- 对话主区
- 追问建议区
- 焦点对象摘要区
- proposal 入口区

依赖数据：

- messages
- latestAnswer
- selectedImpact
- quickPrompts
- pending proposals

验收标准：

- 可从全局入口或对象入口打开
- 可自动带入当前对象上下文
- 可继续追问并返回结构化回答

## 3.3 P1：行动处置页

目标：

- 承接 proposal 审批、分析包查看和执行编排。

建议拆分模块：

- 执行流程板
- proposal 队列
- 区域分析包历史
- 工具调用摘要
- 时间线区

## 3.4 P1：智能体协同页

目标：

- 让用户理解智能体当前在做什么、为什么这样做。

建议拆分模块：

- 智能体任务时间线
- 当前任务状态
- 协同关系摘要
- open questions / blocked reasons
- supervisor 状态区

## 3.5 P2：分众预警与审计增强

目标：

- 完成通知内容对比、证据链查看和审计解释。

建议拆分模块：

- 分众消息对比区
- 审计问题提示区
- 证据链抽屉

## 4. 组件拆解

建议新增或重构组件：

- `DigitalTwinOverviewScreen`
- `CityTwinCanvas`
- `CommandRibbon`
- `RiskSignalBand`
- `DecisionRail`
- `FocusObjectRibbon`
- `AgentCommandDialog`
- `AgentControlConsole`
- `ImpactLinkDrawer`
- `ScenarioCompareBoard`

## 5. 状态拆解

建议按以下状态组织前端实现：

- 全局事件状态
- 数字孪生画布聚焦状态
- 当前选中对象状态
- 智能体对话框开关状态
- proposal 队列状态
- 实时更新状态

其中关键状态包括：

- `selectedImpact`
- `messages`
- `latestAnswer`
- `pendingProposals`
- `agentStatus`

## 6. 接口对接优先级

### 第一批

- 当前 `V2` 事件、对象、proposal、消息流接口

### 第二批

- `V3` impact chain
- `V3` scenario
- `V3` agent council

### 第三批

- `V3` warnings
- `V3` postmortem

## 7. 推荐开发顺序

1. 重构首页骨架为总览大屏。
2. 接入对象联动与右侧决策带。
3. 增加智能体对话控制弹窗。
4. 打通主屏到审批的跳转和回写。
5. 重构行动处置页。
6. 重构智能体协同页。
7. 增强分众预警和审计复盘表现。

## 8. 阶段验收建议

### 阶段一

- 首页具备数字孪生主屏形态
- 可查看重点对象
- 可打开智能体对话控制

### 阶段二

- proposal 审批链路可从主屏打通
- 行动处置页可承接会商与执行流程

### 阶段三

- 分众预警、审计解释和复盘表现完整
- 前端整体风格统一

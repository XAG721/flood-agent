# AgentTwin Flood V3 知识图谱与本体设计

## 1. 设计目标

知识图谱用于把“天气事件、空间区域、重点对象、脆弱性、影响后果、动作模板、责任部门、资源约束”连成可推理的关系网，支撑影响链生成与证据追踪。

## 2. 核心本体

建议定义以下实体：

- `Hazard`
- `Area`
- `Object`
- `Vulnerability`
- `Impact`
- `SecondaryImpact`
- `Action`
- `Department`
- `Resource`
- `Audience`
- `Message`
- `ApprovalLevel`

## 3. 关键关系

| 主体 | 关系 | 客体 |
|---|---|---|
| Hazard | affects_area | Area |
| Hazard | threatens_object | Object |
| Object | has_vulnerability | Vulnerability |
| Vulnerability | amplifies | Impact |
| Impact | may_trigger | SecondaryImpact |
| Impact | suggests_action | Action |
| Action | owned_by | Department |
| Action | consumes_resource | Resource |
| Action | requires_approval | ApprovalLevel |
| Message | targets_audience | Audience |

## 4. 三元组样例

```text
强降雨 -> threatens_object -> 东关南街下穿通道
东关南街下穿通道 -> has_vulnerability -> 地势低洼
地势低洼 -> amplifies -> 车辆滞留
车辆滞留 -> suggests_action -> 交通巡查与临时警戒
交通巡查与临时警戒 -> owned_by -> 交警
交通巡查与临时警戒 -> requires_approval -> commander
```

## 5. 数据来源

- `v2_entity_profiles` 中的对象画像
- `rag_raw/policy` 中的政策与规则
- `rag_raw/case` 中的历史案例
- `rag_raw/profile` 中的场景画像
- 现有资源状态与执行日志

## 6. 抽取规则

### 6.1 规则抽取

从政策/SOP 中抽取：

- 触发条件
- 禁止动作
- 审批动作
- 责任部门

### 6.2 案例抽取

从案例中抽取：

- 受影响对象
- 影响后果
- 有效动作
- 失败动作

### 6.3 对象画像抽取

从对象画像中抽取：

- 对象类型
- 位置
- 脆弱性
- 关键资产
- 通知偏好

## 7. 使用方式

### 7.1 对影响链的支撑

输入：

- 天气事件
- 影响区域

输出：

- 可能受影响对象
- 影响原因
- 推荐动作

### 7.2 对审计的支撑

每条建议都应能回溯到：

- 哪条规则
- 哪条案例
- 哪个对象画像

## 8. 人工审核规则

以下内容不允许自动入图，必须人工审核：

- 新地名
- 新部门
- 新审批级别
- 高风险动作与其责任边界

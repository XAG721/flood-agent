# AgentTwin Flood V3 测试与评测方案

## 1. 测试目标

确保 `V3` 升级后的系统满足：

- 功能可用
- 结构化输出稳定
- 证据可追溯
- 权限边界有效
- 分众消息一致

## 2. 测试分层

## 2.1 单元测试

覆盖模块：

- `weather_event_parser`
- `impact_chain_engine`
- `action_scorer`
- `message_consistency_checker`
- `approval_policy`

## 2.2 集成测试

覆盖链路：

1. 天气事件解析
2. 影响链生成
3. 情景方案生成
4. 会商输出
5. proposal 生成
6. 分众消息生成

## 2.3 前后端联调测试

验证：

- REST 接口正确
- SSE 实时更新正常
- 审批后状态回写正常

## 2.4 智能体评测

重点评测：

- 是否引用证据
- 是否出现不存在的地点/资源
- 是否越权
- 是否给出冲突消息

## 3. 验收场景

建议至少准备以下样例：

1. 正常强降雨预警样例
2. 对象画像缺失样例
3. 资源不足样例
4. 需要人工审批样例
5. 地点不存在样例
6. 分众消息冲突样例

## 4. 核心用例

### 用例 1：天气文本解析

- 输入：一段强降雨预警文本
- 期望：返回结构化天气事件

### 用例 2：影响链生成

- 输入：事件 + 对象画像 + RAG
- 期望：至少生成一条包含对象、影响、动作、证据的影响链

### 用例 3：多方案推演

- 输入：完整影响链
- 期望：返回至少三套方案和排序结果

### 用例 4：审计阻断

- 输入：包含虚构资源的方案
- 期望：AuditAgent 阻断 proposal 输出

### 用例 5：审批闭环

- 输入：待审批 proposal
- 期望：审批成功后生成通知草稿、执行日志和审计记录

### 用例 6：分众消息一致性

- 输入：同一 proposal 的四类受众消息
- 期望：风险等级、区域和时窗一致

## 5. 质量指标

建议建立以下指标：

| 指标 | 说明 |
|---|---|
| `impact_chain_completeness` | 影响链字段完整率 |
| `scenario_rank_stability` | 同类输入的方案排序稳定性 |
| `evidence_coverage_rate` | 建议中包含证据引用的比例 |
| `hallucination_rate` | 虚构对象/资源/政策比例 |
| `approval_boundary_pass_rate` | 审批边界测试通过率 |
| `message_consistency_pass_rate` | 分众消息一致性通过率 |

## 6. 自动化建议

### 后端

- `pytest` 覆盖引擎与接口
- 构造固定种子数据

### 前端

- 组件测试覆盖审批弹窗、方案对比、分众消息面板
- 关键链路做端到端冒烟测试

## 7. 演示验收脚本

建议固定一条答辩演示脚本：

1. 输入强降雨预警文本
2. 展示影响链
3. 展示三套方案
4. 展示会商输出
5. 审批 proposal
6. 生成四类预警消息
7. 查看审计记录和复盘入口

# AgentTwin Flood AgentTwin 智能体设计

## 1. 设计原则

1. 每个智能体只负责有限职责。
2. 所有智能体输出必须结构化。
3. 高风险动作必须由 `AuditAgent + 人工审批` 双重门禁控制。
4. 智能体输出不仅服务于后端决策，也必须能够被前端的数字孪生主屏和智能体对话控制台直接消费。

## 2. 智能体列表

## 2.1 Orchestrator

- 职责：分发上下文、调度执行顺序、汇总最终输出。
- 输入：事件、影响链、候选方案、角色输出。
- 输出：会商结论、最终 recommendation、待审批 proposal。
- 前端关系：负责形成能在“总览大屏右侧决策带”和“智能体对话控制台”中展示的统一摘要。
- 禁止行为：直接绕过其他角色生成最终高风险动作。

## 2.2 WeatherAgent

- 职责：解释天气文本和预警窗口。
- 输入：原始天气文本、区域信息。
- 输出：结构化天气事件。
- 工具：天气解析器、区域匹配器。
- 前端关系：其输出应能在总览大屏顶部事件条和左侧态势带中直接显示。
- 禁止行为：直接生成执行动作。

## 2.3 ImpactAgent

- 职责：识别对象影响与次生影响。
- 输入：天气事件、对象画像、RAG 证据。
- 输出：影响链列表。
- 工具：知识图谱、RAG 检索。
- 前端关系：其输出应可驱动数字孪生主屏中的风险热区、对象联动和影响链抽屉。
- 禁止行为：虚构对象或地点。

## 2.4 ActionAgent

- 职责：生成候选行动方案。
- 输入：影响链、动作模板、约束。
- 输出：`scenario` 列表。
- 工具：动作模板库、评分器。
- 前端关系：其输出应可在行动处置页和智能体对话控制台中以“候选方案 + 取舍说明”形式展示。
- 禁止行为：绕过审批边界指定强制执行。

## 2.5 ResourceAgent

- 职责：校验资源可用性与执行可行性。
- 输入：候选方案、资源状态。
- 输出：资源约束结论。
- 工具：资源状态查询。
- 前端关系：其输出应能支持主屏中的资源节点状态和行动页中的资源约束说明。
- 禁止行为：虚构资源存在。

## 2.6 CommunicationAgent

- 职责：生成分众预警内容。
- 输入：已审批 proposal、Alert Object。
- 输出：领导版、部门版、社区版、公众版消息。
- 工具：消息模板库、一致性检查器。
- 前端关系：其输出应可在分众预警对比界面中并列展示。
- 禁止行为：夸大公众风险或更改审批结论。

## 2.7 AuditAgent

- 职责：执行证据、权限和一致性校验。
- 输入：所有角色输出。
- 输出：`audit_result`。
- 工具：证据校验器、权限策略、消息一致性检查器。
- 前端关系：其结果应直接影响 proposal 是否能进入待审批区，并能在审计复盘页中解释阻断原因。
- 禁止行为：修改业务事实，只能拦截、标注和解释问题。

## 3. 统一输出结构

每个智能体统一输出：

```json
{
  "agent_name": "impact_agent",
  "summary": "识别出 3 个重点受影响对象",
  "confidence": 0.83,
  "evidence_refs": ["policy_x", "case_y"],
  "structured_output": {},
  "missing_slots": [],
  "recommended_next_tasks": []
}
```

补充要求：

1. `summary` 应能直接被前端卡片、时间线和对话控制台引用。
2. `structured_output` 应服务于图层、对象和方案面板联动。
3. `recommended_next_tasks` 应能映射为前端的继续追问建议或下一步动作。

## 4. 智能体编排顺序

推荐主链路：

1. WeatherAgent
2. ImpactAgent
3. ActionAgent
4. ResourceAgent
5. CommunicationAgent
6. AuditAgent
7. Orchestrator 汇总

## 5. 前端交互映射

为了匹配“数字孪生智能体洪水预警系统前端”，建议建立以下映射关系：

| 智能体 | 前端主要承接位置 |
|---|---|
| WeatherAgent | 顶部事件条、左侧态势带 |
| ImpactAgent | 中央数字孪生画布、影响链抽屉 |
| ActionAgent | 行动处置页、对话控制台中的方案比较区 |
| ResourceAgent | 资源状态面板、执行约束说明区 |
| CommunicationAgent | 分众预警对比面板 |
| AuditAgent | proposal 审批阻断提示、审计复盘页 |
| Orchestrator | 右侧决策带、全局智能体摘要 |

## 6. 失败处理

### 6.1 缺失信息

- 标注 `missing_slots`
- 降级为保守建议
- 不生成高风险动作

### 6.2 校验失败

- 进入 `audit_blocked`
- 给出阻断原因
- 转入人工处理

## 7. 与当前 现有平台 的关系

当前 `flood_system/platform/multi_agent.py` 已存在：

- `HazardAgent`
- `ExposureAgent`
- `ResourceAgent`
- `PlanningAgent`
- `PolicyAgent`
- `CommsAgent`

`AgentTwin` 可视为在此基础上的角色语义升级：

- `HazardAgent -> WeatherAgent`
- `ExposureAgent -> ImpactAgent`
- `PlanningAgent -> ActionAgent`
- `PolicyAgent -> AuditAgent / ApprovalPolicy`
- `CommsAgent -> CommunicationAgent`

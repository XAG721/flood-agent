# AgentTwin Flood V3 智能体设计

## 1. 设计原则

1. 每个智能体只负责有限职责。
2. 所有智能体输出必须结构化。
3. 高风险动作必须由 `AuditAgent + 人工审批` 双重门禁控制。

## 2. 智能体列表

## 2.1 Orchestrator

- 职责：分发上下文、调度执行顺序、汇总最终输出
- 输入：事件、影响链、候选方案、角色输出
- 输出：会商结论、最终 recommendation、待审批 proposal
- 禁止行为：直接绕过其他角色生成最终高风险动作

## 2.2 WeatherAgent

- 职责：解释天气文本和预警窗口
- 输入：原始天气文本、区域信息
- 输出：结构化天气事件
- 工具：天气解析器、区域匹配器
- 禁止行为：直接生成执行动作

## 2.3 ImpactAgent

- 职责：识别对象影响与次生影响
- 输入：天气事件、对象画像、RAG 证据
- 输出：影响链列表
- 工具：知识图谱、RAG 检索
- 禁止行为：虚构对象或地点

## 2.4 ActionAgent

- 职责：生成候选行动方案
- 输入：影响链、动作模板、约束
- 输出：`scenario` 列表
- 工具：动作模板库、评分器
- 禁止行为：绕过审批边界指定强制执行

## 2.5 ResourceAgent

- 职责：校验资源可用性与执行可行性
- 输入：候选方案、资源状态
- 输出：资源约束结论
- 工具：资源状态查询
- 禁止行为：虚构资源存在

## 2.6 CommunicationAgent

- 职责：生成分众预警内容
- 输入：已审 proposal、Alert Object
- 输出：领导版/部门版/社区版/公众版消息
- 工具：消息模板库、一致性检查器
- 禁止行为：夸大公众风险或更改审批结论

## 2.7 AuditAgent

- 职责：执行证据、权限和一致性校验
- 输入：所有角色输出
- 输出：`audit_result`
- 工具：证据校验器、权限策略、消息一致性检查
- 禁止行为：修改业务事实，只能拦截或标注问题

## 3. 建议输出结构

每个智能体统一输出：

```json
{
  "agent_name": "impact_agent",
  "summary": "识别出 3 个重点受影响对象",
  "confidence": 0.83,
  "evidence_refs": ["policy_x", "case_y"],
  "structured_output": {},
  "missing_slots": [],
  "recommended_next_tasks": []
}
```

## 4. 智能体编排顺序

推荐主链路：

1. WeatherAgent
2. ImpactAgent
3. ActionAgent
4. ResourceAgent
5. CommunicationAgent
6. AuditAgent
7. Orchestrator 汇总

## 5. 失败处理

### 5.1 缺失信息

- 标注 `missing_slots`
- 降级为保守建议
- 不生成高风险动作

### 5.2 校验失败

- 进入 `audit_blocked`
- 给出阻断原因
- 转入人工处理

## 6. 与现有 V2 的关系

当前 `flood_system/v2/multi_agent.py` 已经存在：

- `HazardAgent`
- `ExposureAgent`
- `ResourceAgent`
- `PlanningAgent`
- `PolicyAgent`
- `CommsAgent`

`V3` 可以视为在此基础上做角色语义升级：

- `HazardAgent -> WeatherAgent`
- `ExposureAgent -> ImpactAgent`
- `PlanningAgent -> ActionAgent`
- `PolicyAgent -> AuditAgent / ApprovalPolicy`
- `CommsAgent -> CommunicationAgent`

# AgentTwin Flood V3 产品需求说明

## 1. 产品背景

当前系统已经实现 `V2` 级的“主动预警 + 主动请示 + 人工审批 + 执行留痕”闭环，但核心表达仍以风险等级、对象建议和区域 proposal 为主。根据 [`AgentTwin-Flood-Requirements.md`](../../AgentTwin-Flood-Requirements.md)，系统下一阶段要升级为以“影响链、行动推演、多智能体会商、分众预警、可审计 proposal”为核心的辅助决策系统。

产品目标不是替代人的指挥判断，而是把“天气预警”稳定转换为“影响对象”和“可执行行动”，再把结果转换为不同角色可直接使用的信息。

## 2. 产品目标

### 2.1 总目标

把系统从“洪水风险展示与建议生成平台”升级为“面向城市内涝场景的 AgentTwin 辅助决策平台”。

### 2.2 本期目标

1. 从天气/人工文本输入中生成结构化天气事件。
2. 从事件生成影响链，识别受影响对象、影响后果和证据来源。
3. 基于影响链推演多个候选行动方案并排序。
4. 通过多智能体分工会商输出可审计 proposal。
5. 生成领导版、部门版、社区版、公众版等分众预警内容。
6. 保留人在回路审批和全过程留痕。

### 2.3 非目标

1. 不把高精度水动力预测作为本期核心。
2. 不直接接入生产级短信网关或政务审批系统。
3. 不允许模型绕过审批直接执行高风险动作。
4. 不在本期实现传感器大规模实时接入或视频识别。

## 3. 用户角色

### 3.1 业务角色

- 指挥员：审批高风险 proposal，查看会商结果和分众预警草稿。
- 值班员：录入事件、查看态势、发起补充分析、跟进执行。
- 专业部门人员：查看部门任务清单和依据。
- 社区/街道人员：接收巡查清单、居民提醒和执行反馈。
- 公众：接收简洁一致的风险提醒。

### 3.2 系统角色

- Orchestrator：编排多智能体流程。
- WeatherAgent：解释天气事件。
- ImpactAgent：生成影响链。
- ActionAgent：推演行动方案。
- ResourceAgent：校验资源与约束。
- CommunicationAgent：生成分众消息。
- AuditAgent：校验证据、权限和一致性。

## 4. 核心场景

### 场景一：强降雨事件自动解析

用户输入一段气象预警文本，系统自动提取：

- 灾害类型
- 影响区域
- 时间窗口
- 预警等级
- 确定性

### 场景二：影响链推演

系统将天气事件与对象画像、历史案例、SOP、RAG 文档关联，输出：

- 受影响对象
- 主要影响
- 次生影响
- 证据链
- 置信度

### 场景三：多方案行动推演

系统围绕同一事件生成多套方案，例如：

- 标准防御方案
- 资源优先方案
- 脆弱群体优先方案

并给出排序理由。

### 场景四：多智能体会商与 proposal 生成

系统将结构化输入分发给各角色智能体，产出：

- 角色结论
- 会商摘要
- 审计结果
- 待审批 proposal

### 场景五：分众预警发布

同一 proposal 被改写为：

- 领导摘要
- 部门任务单
- 社区巡查提醒
- 公众提示

并做一致性校验。

## 5. 功能范围

## 5.1 后端功能

- `weather-event parser`
- `impact chain engine`
- `scenario decision lab`
- `agent council`
- `audience warning center`
- `evidence verifier`
- `approval policy`
- `decision memory`

## 5.2 前端功能

- 影响态势大屏
- 影响链面板
- 会商面板
- 情景推演面板
- proposal 审批弹窗
- 分众预警面板
- 证据链抽屉
- 复盘页面

## 5.3 数据功能

- 对象画像维护
- 动作模板维护
- 资源约束维护
- RAG 文档管理
- 历史案例与复盘沉淀

## 6. 关键页面需求

### 6.1 总览页

展示事件概况、影响地图、重点对象、风险趋势、待审批事项。

### 6.2 影响链页

按“天气 -> 对象 -> 影响 -> 行动”展示链路，并可查看证据来源。

### 6.3 会商与推演页

展示角色智能体观点、冲突点、候选方案、评分和排序。

### 6.4 proposal 审批页

展示：

- 建议动作
- 对应对象
- 责任部门
- 执行时限
- 审批等级
- 证据链

### 6.5 分众预警页

同屏查看多个受众版本，支持一致性差异高亮。

## 7. 验收标准

1. 能从天气文本生成结构化天气事件。
2. 能生成至少一条完整影响链。
3. 能生成至少三套候选行动方案并排序。
4. 能形成一份包含证据和审批边界的 proposal。
5. 能输出至少四类分众预警内容。
6. 高风险动作必须经人工审批。
7. 全过程可追溯到输入、证据、模型、操作人与审批人。
